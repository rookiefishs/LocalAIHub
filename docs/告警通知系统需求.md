# 告警通知系统需求文档

## 概述

本文档描述 LocalAIHub 告警通知功能的需求，用于监控服务状态并及时通知管理员。

## 功能列表

### 1. 告警规则管理

#### 1.1 告警类型

| 类型 | 描述 | 触发条件 |
|------|------|----------|
| request_failure_rate | 请求失败率告警 | 失败率超过阈值 |
| api_key_quota | API Key 配额告警 | 配额使用超过阈值 |
| provider_unavailable | 上游服务不可用 | 连续失败超过阈值 |
| circuit_breaker_open | 熔断开启告警 | 熔断器状态变为 open |

#### 1.2 告警级别

| 级别 | 描述 | 颜色标识 |
|------|------|----------|
| critical | 严重 | 红色 |
| warning | 警告 | 黄色 |
| info | 信息 | 蓝色 |

### 2. 通知渠道

#### 2.1 支持的通知方式

- 邮件通知
- 钉钉 Webhook
- 企业微信 Webhook

#### 2.2 渠道配置

```sql
CREATE TABLE IF NOT EXISTS alert_channel (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(64) NOT NULL,
    channel_type VARCHAR(16) NOT NULL COMMENT 'email/dingtalk/wechat',
    config_json JSON NOT NULL COMMENT '渠道配置',
    enabled TINYINT(1) NOT NULL DEFAULT 1,
    created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    updated_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3. 告警规则配置

```sql
CREATE TABLE IF NOT EXISTS alert_rule (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(64) NOT NULL,
    alert_type VARCHAR(32) NOT NULL,
    condition_json JSON NOT NULL,
    threshold DECIMAL(10,4) NOT NULL,
    check_interval_seconds INT NOT NULL DEFAULT 60,
    enabled TINYINT(1) NOT NULL DEFAULT 1,
    channel_ids JSON NOT NULL,
    created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    updated_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 4. 告警记录

```sql
CREATE TABLE IF NOT EXISTS alert_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    rule_id BIGINT NOT NULL,
    alert_type VARCHAR(32) NOT NULL,
    level VARCHAR(16) NOT NULL,
    title VARCHAR(128) NOT NULL,
    message TEXT NOT NULL,
    triggered_at DATETIME(3) NOT NULL,
    resolved_at DATETIME(3) NULL,
    status VARCHAR(16) NOT NULL DEFAULT 'pending',
    created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    KEY idx_alert_record_rule (rule_id),
    KEY idx_alert_record_status (status),
    KEY idx_alert_record_triggered (triggered_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 后端 API 设计

### 告警规则

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /admin/api/v1/alert/rules | 获取告警规则列表 |
| POST | /admin/api/v1/alert/rules | 创建告警规则 |
| PUT | /admin/api/v1/alert/rules/{id} | 更新告警规则 |
| DELETE | /admin/api/v1/alert/rules/{id} | 删除告警规则 |
| PUT | /admin/api/v1/alert/rules/{id}/status | 启用/禁用规则 |

### 通知渠道

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /admin/api/v1/alert/channels | 获取通知渠道列表 |
| POST | /admin/api/v1/alert/channels | 创建通知渠道 |
| PUT | /admin/api/v1/alert/channels/{id} | 更新通知渠道 |
| DELETE | /admin/api/v1/alert/channels/{id} | 删除通知渠道 |

### 告警记录

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /admin/api/v1/alert/records | 获取告警记录 |
| PUT | /admin/api/v1/alert/records/{id}/resolve | 标记为已处理 |

## 前端页面设计

入口：系统管理 → 告警配置

包含三个 Tab：
- 告警规则
- 通知渠道
- 告警记录

### 2. 告警规则 Tab

- 规则列表（表格）
- 创建规则按钮
- 编辑/删除/启用禁用操作

### 3. 通知渠道 Tab

- 渠道列表
- 添加渠道按钮
- 测试发送按钮
