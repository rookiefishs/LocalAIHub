# 操作审计日志需求文档

## 概述

本文档描述 LocalAIHub 操作审计日志功能的需求，用于记录所有管理员操作并支持查询和导出。

## 功能列表

### 1. 审计范围

记录以下操作类型：

| 操作类型 | 描述 | 涉及模块 |
|----------|------|----------|
| create | 创建资源 | 上游、模型、路由、API Key |
| update | 更新资源 | 上游、模型、路由、API Key |
| delete | 删除资源 | 上游、模型、路由、API Key |
| enable | 启用资源 | 上游、模型、路由、API Key |
| disable | 禁用资源 | 上游、模型、路由、API Key |
| lock | 锁定路由 | 路由管理 |
| unlock | 解锁路由 | 路由管理 |
| login | 管理员登录 | 系统 |
| logout | 管理员登出 | 系统 |
| export | 导出配置 | 系统 |
| import | 导入配置 | 系统 |

### 2. 审计内容

每条审计记录包含：

- 操作时间
- 操作人（管理员 ID 和用户名）
- 操作类型
- 操作对象类型和 ID
- 变更内容（JSON 格式）
- IP 地址
- User-Agent

### 3. 现有表结构

项目已有 `audit_log` 表，直接使用：

```sql
CREATE TABLE IF NOT EXISTS audit_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    admin_user_id BIGINT NOT NULL,
    action VARCHAR(64) NOT NULL,
    target_type VARCHAR(64) NOT NULL,
    target_id BIGINT NULL,
    change_summary_json JSON NULL,
    ip_address VARCHAR(64) NULL,
    user_agent VARCHAR(255) NULL,
    request_id VARCHAR(64) NULL,
    created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    KEY idx_audit_log_admin_user_created_at (admin_user_id, created_at),
    KEY idx_audit_log_target (target_type, target_id),
    KEY idx_audit_log_action_created_at (action, created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 后端 API 设计

### 审计日志

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /admin/api/v1/audit-logs | 获取审计日志列表 |
| GET | /admin/api/v1/audit-logs/{id} | 获取单条日志详情 |
| GET | /admin/api/v1/audit-logs/export | 导出 CSV |

### API 详情

#### GET /admin/api/v1/audit-logs

**请求参数**

| 参数 | 类型 | 必填 | 描述 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| page_size | int | 否 | 每页数量，默认 20 |
| start_date | string | 否 | 开始日期，格式 YYYY-MM-DD |
| end_date | string | 否 | 结束日期，格式 YYYY-MM-DD |
| admin_user_id | int | 否 | 管理员 ID |
| action | string | 否 | 操作类型 |
| target_type | string | 否 | 目标类型 |
| keyword | string | 否 | 关键词搜索 |

**响应示例**

```json
{
    "code": 0,
    "data": {
        "items": [
            {
                "id": 1,
                "admin_user_id": 1,
                "admin_username": "admin",
                "action": "create",
                "target_type": "provider",
                "target_id": 5,
                "target_name": "OpenAI",
                "change_summary": {
                    "name": "OpenAI",
                    "base_url": "https://api.openai.com"
                },
                "ip_address": "192.168.1.100",
                "created_at": "2024-01-01T10:00:00Z"
            }
        ],
        "total": 100,
        "page": 1,
        "page_size": 20
    }
}
```

#### GET /admin/api/v1/audit-logs/export

**请求参数**

同列表接口，导出全部或筛选后的数据。

**响应**

返回 CSV 文件下载。

## 前端页面设计

### 1. 入口

侧边栏添加"审计日志"菜单项，位于"日志中心"下方。

### 2. 页面布局

```
┌─────────────────────────────────────────────────────────────┐
│ 审计日志                                    [导出 CSV ]   │
├─────────────────────────────────────────────────────────────┤
│ [时间范围 ▼] [操作类型 ▼] [操作对象 ▼] [操作人 ▼] [搜索]    │
├─────────────────────────────────────────────────────────────┤
│ 时间          │ 操作人 │ 操作类型 │ 操作对象  │ 详情        │
│───────────────┼────────┼──────────┼───────────┼─────────────│
│ 2024-01-01    │ admin  │ create   │ provider  │ 创建上游xxx  │
│ 10:00:00      │        │          │           │             │
│───────────────┼────────┼──────────┼───────────┼─────────────│
│ 2024-01-01    │ admin  │ update   │ api_key   │ 更新Key xxx │
│ 09:30:00      │        │          │           │             │
├─────────────────────────────────────────────────────────────┤
│                                                [< 1 2 3 >] │
└─────────────────────────────────────────────────────────────┘
```

### 3. 表格列

| 列名 | 描述 |
|------|------|
| 时间 | 操作时间，精确到秒 |
| 操作人 | 管理员用户名 |
| 操作类型 | create/update/delete/enable/disable 等 |
| 操作对象 | 类型 + 名称/ID |
| 详情 | 变更内容摘要 |

### 4. 功能说明

- 点击行可展开查看详情（JSON 格式）
- 导出按钮下载 CSV 文件
- 支持时间范围选择
