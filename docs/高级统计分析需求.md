# 高级统计分析需求文档

## 概述

本文档描述 LocalAIHub 高级统计分析功能的需求，包括成本统计和对比分析。

## 功能列表

### 1. 成本统计

#### 1.1 需求描述

- 按供应商统计 Token 消耗（prompt/completion/total）
- 按模型统计 Token 消耗
- 支持配置各模型的单价（每千 tokens 费用）
- 自动计算预估费用

#### 1.2 数据库变更

```sql
-- 新增模型单价配置表
CREATE TABLE IF NOT EXISTS model_pricing (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    model_code VARCHAR(128) NOT NULL,
    provider_id BIGINT NULL,
    prompt_price_per_1k DECIMAL(10,6) NOT NULL DEFAULT 0,
    completion_price_per_1k DECIMAL(10,6) NOT NULL DEFAULT 0,
    currency VARCHAR(8) NOT NULL DEFAULT 'CNY',
    enabled TINYINT(1) NOT NULL DEFAULT 1,
    remark VARCHAR(255) NULL,
    created_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    updated_at DATETIME(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    UNIQUE KEY uk_model_pricing_model (model_code, provider_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 成本统计视图（可选）
CREATE OR REPLACE VIEW v_provider_cost AS
SELECT 
    p.id AS provider_id,
    p.name AS provider_name,
    DATE(rl.created_at) AS date,
    SUM(rl.prompt_tokens) AS prompt_tokens,
    SUM(rl.completion_tokens) AS completion_tokens,
    SUM(rl.total_tokens) AS total_tokens,
    COUNT(*) AS request_count
FROM request_log rl
LEFT JOIN provider p ON rl.provider_id = p.id
WHERE rl.created_at >= DATE_SUB(UTC_TIMESTAMP(), INTERVAL 30 DAY)
GROUP BY p.id, DATE(rl.created_at);
```

### 2. 对比分析

#### 2.1 需求描述

- 与前一天/上周/上月同期对比
- 计算增长率（请求数、Token、费用）
- 显示上升/下降箭头和百分比

#### 2.2 后端 API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /admin/api/v1/analytics/comparison | 同比环比数据 |

**响应示例**

```json
{
    "code": 0,
    "data": {
        "request_count": {

            "previous": 1100,
            "change": 134,
```
